## Экран отчётов

По умолчанию открывается за текущий месяц.
Переключение периода — редактирует то же сообщение.
Активный период помечается точкой в кнопке: `· Месяц`

Формат:
```
📊 Отчёты — Март 2026
━━━━━━━━━━━━━━━
💰 Выручка: 48 500 ₽
🛒 Заказов выполнено: 23
👥 Новых клиентов: 4
🔄 Повторных клиентов: 19
🧾 Средний чек: 2 108 ₽
📋 Всего клиентов в базе: 87
━━━━━━━━━━━━━━━
🏆 Топ услуг по выручке:
- Уборка квартиры — 28 500 ₽ (12 раз)
- Мойка окон — 12 000 ₽ (8 раз)
- Химчистка дивана — 8 000 ₽ (3 раза)
━━━━━━━━━━━━━━━
[· Месяц] [Сегодня] [Неделя] [📅 Период]
[🏠 Главная]
```

Если данных за период нет:
```
За этот период заказов нет.
```
(остальные показатели показывать как 0, топ услуг не показывать)

## Периоды

| Кнопка | Диапазон |
|---|---|
| Сегодня | today 00:00 — today 23:59 |
| Неделя | today - 6 days 00:00 — today 23:59 |
| Месяц | first day of current month — today 23:59 |
| Период | произвольный, через FSM |

**FSM произвольного периода** (`ReportPeriodFSM`):

Шаг 1 — дата начала:
```
Введите дату начала периода
(например: 01.03.2026):
[❌ Отмена]
```

Шаг 2 — дата конца:
```
Введите дату окончания периода
(например: 31.03.2026):
[❌ Отмена]
```

Валидация:
- Дата конца не раньше даты начала
- Обе даты парсятся через существующую `parse_date()`
- Период не больше 1 года (защита от тяжёлых запросов)

После ввода — показать отчёт с заголовком:
```
📊 Отчёты — 01.03 — 31.03.2026
```

## Расчёт показателей

Все данные берутся из заказов со статусом `done` за указанный период (`done_at`).
```python
# Выручка
SELECT SUM(amount_total) FROM orders
WHERE master_id = ? AND status = 'done'
AND done_at BETWEEN ? AND ?

# Количество выполненных заказов
SELECT COUNT(*) FROM orders
WHERE master_id = ? AND status = 'done'
AND done_at BETWEEN ? AND ?

# Новые клиенты за период
SELECT COUNT(*) FROM master_clients
WHERE master_id = ?
AND first_visit BETWEEN ? AND ?

# Повторные клиенты (были до начала периода, заказ в периоде)
SELECT COUNT(DISTINCT client_id) FROM orders
WHERE master_id = ? AND status = 'done'
AND done_at BETWEEN ? AND ?
AND client_id IN (
    SELECT client_id FROM orders
    WHERE master_id = ? AND status = 'done'
    AND done_at < ?  -- раньше начала периода
)

# Средний чек
выручка / количество заказов (если > 0)

# Всего клиентов в базе (не за период — общее)
SELECT COUNT(*) FROM master_clients WHERE master_id = ?

# Топ-5 услуг по выручке
SELECT name, SUM(price) as revenue, COUNT(*) as cnt
FROM order_items
WHERE order_id IN (
    SELECT id FROM orders
    WHERE master_id = ? AND status = 'done'
    AND done_at BETWEEN ? AND ?
)
GROUP BY name
ORDER BY revenue DESC
LIMIT 5
```

## Критерии проверки

1. Отчёт за текущий месяц открывается по умолчанию с реальными данными
2. Переключение Сегодня/Неделя/Месяц обновляет все показатели
3. Активный период помечен точкой в кнопке
4. FSM произвольного периода — оба шага работают, валидация дат
5. Топ-5 услуг показывает выручку И количество заказов
6. Средний чек считается корректно (не делить на 0)
7. Новые и повторные клиенты считаются без пересечений
8. Если данных нет — показывает нули, топ услуг скрыт
9. Период больше 1 года — валидация с понятным сообщением
10. Заголовок отчёта отражает выбранный период
