## Часть 1 — Настройки мастера

### 1.1 Профиль мастера

Экран показывает текущие значения всех полей и кнопку для каждого поля.
После редактирования любого поля — текст блока обновляется сразу (edit_text того же сообщения).

Формат экрана:
```
👤 Профиль
━━━━━━━━━━━━━━━
Имя: Анна Мастерова
Телефон: +7 999 123-45-67
Сайт: anna-clean.ru
Соцсети: instagram.com/anna_clean
Контакты: @anna_clean
Режим работы: пн-пт 9:00-19:00
━━━━━━━━━━━━━━━
[✏️ Имя]         [✏️ Телефон]
[✏️ Сайт]        [✏️ Соцсети]
[✏️ Контакты]    [✏️ Режим работы]
[📅 Google Calendar]
[◀️ Назад]  [🏠 Главная]
```

Если поле не заполнено — показывать «не указано».

Поведение при нажатии на кнопку поля:
1. Бот запрашивает новое значение текстом
2. Мастер вводит
3. Значение сохраняется в БД
4. Экран профиля обновляется с новым значением

FSM `ProfileEditFSM`:
- `waiting_field` — ожидание выбора поля (через callback)
- `waiting_value` — ожидание ввода нового значения

Кнопка `[📅 Google Calendar]` — заглушка (реализуем позже).

### 1.2 Бонусная программа

Экран показывает текущие настройки и позволяет редактировать каждый параметр по той же схеме: нажал кнопку → ввёл → обновилось.

Формат экрана:
```
🎁 Бонусная программа
━━━━━━━━━━━━━━━
Статус: ✅ Включена
Начисление: 5% от суммы заказа
Макс. списание: 50% суммы заказа
Бонус на день рождения: 300 ₽
━━━━━━━━━━━━━━━
[🔄 Вкл / Выкл]
[✏️ % начисления]    [✏️ % списания]
[✏️ Бонус на ДР]
[◀️ Назад]  [🏠 Главная]
```

Валидация:
- % начисления: число от 0 до 100
- % списания: число от 0 до 100
- Бонус на ДР: положительное целое число
- Кнопка «Вкл/Выкл» — переключает `bonus_enabled` без ввода, сразу обновляет экран

## Часть 2 — Блок Клиенты

### 2.1 Карточка клиента

Реализовать полный функционал карточки.

Формат:
```
👤 Иван Петров
📞 +7 999 123-45-67
🎂 15 марта 1985
💰 Бонусов: 450 ₽
🛒 Заказов: 8 | Потрачено: 24 500 ₽
📝 Заметка: аллергия на хлорку
━━━━━━━━━━━━━━━
[📋 История]      [🎁 Бонусы]
[✏️ Редактировать] [📝 Заметка]

Статистику (кол-во заказов, потрачено) считать из БД в реальном времени.
Если заметки нет — показывать «📝 Заметка: —».

Та же схема что в профиле мастера: кнопка поля → ввод → обновление экрана.
```

[◀️ Назад]  [🏠 Главная]

После изменения — обновить экран редактирования И данные в карточке клиента.

### 2.3 Заметка о клиенте

Нажал `[📝 Заметка]` → бот запрашивает текст заметки → сохраняет в `master_clients.note` → возвращает в карточку с обновлённой заметкой.

Если заметка уже есть — показать текущую перед запросом:
```
Текущая заметка:
«аллергия на хлорку»

Введите новую заметку или отправьте «-» чтобы удалить:
```

### 2.4 История заказов клиента

Последние 10 заказов, отсортированы по дате (новые сверху).

Формат каждой строки:
```
- 15.02 — Уборка квартиры | 3 500 ₽ | ✅
- 10.01 — Мойка окон | 1 500 ₽ | ✅
- 05.01 — Химчистка | 2 000 ₽ | ❌ отменён
```

Статусы: ✅ выполнен, ❌ отменён, 📅 запланирован.
Кнопки: `[◀️ Назад]  [🏠 Главная]`

### 2.5 Бонусы клиента

Текущий баланс + последние 10 записей лога.

Формат:
```
🎁 Бонусы — Иван Петров
━━━━━━━━━━━━━━━
Баланс: 450 ₽

- 15.02 +175 — за заказ #12
- 10.01 +75 — за заказ #8
- 01.01 +300 — 🎂 день рождения
- 20.12 -200 — списано в заказе #6
━━━━━━━━━━━━━━━
[➕ Начислить]  [➖ Списать]
[◀️ Назад]  [🏠 Главная]
```

**Ручное начисление** (`[➕ Начислить]`):
1. Сумма (положительное целое)
2. Комментарий + `[Пропустить]`
3. Подтверждение → записать в `bonus_log` (тип `manual`), обновить баланс

**Ручное списание** (`[➖ Списать]`):
1. Сумма — валидация: не больше текущего баланса
2. Комментарий + `[Пропустить]`
3. Подтверждение → записать в `bonus_log` (тип `manual`), обновить баланс

### 2.6 Создать заказ из карточки клиента

Кнопка `[➕ Создать заказ]` в карточке клиента — запускает FSM создания заказа с предзаполненным клиентом (шаг выбора клиента пропускается, сразу переходим к адресу).

Передать `client_id` в FSM state перед запуском.

## Критерии проверки

1. Профиль: нажал кнопку поля → ввёл → текст блока обновился с новым значением
2. Бонусная программа: все 4 параметра редактируются, вкл/выкл работает без ввода
3. Карточка клиента: статистика считается из БД (не заглушка)
4. Редактирование клиента: имя, телефон, ДР — все поля меняются
5. Заметка: сохраняется, отображается в карточке, удаляется через «-»
6. История: последние 10 заказов со статусами
7. Бонусы: лог отображается, ручное начисление и списание работают
8. Списание не превышает текущий баланс — валидация
9. Создать заказ из карточки — FSM стартует сразу с шага «Адрес» (клиент уже выбран)
10. Везде кнопки «◀️ Назад» и «🏠 Главная»
