## Часть 1 — Рассылка

### FSM `BroadcastFSM`

**Шаг 1 — Текст сообщения**
```
📨 Новая рассылка

Введите текст сообщения:
[❌ Отмена]
```

**Шаг 2 — Медиа (опционально)**
```
Прикрепить фото или видео? (необязательно)

[Пропустить]  [❌ Отмена]
```
Если мастер отправляет фото/видео — сохранить `file_id`.
Если нажал «Пропустить» — медиа нет.

**Шаг 3 — Сегмент получателей**
```
Кому отправить?

[👥 Всем клиентам]
[😴 Не приходили 3+ месяца]
[💤 Не приходили 6+ месяцев]
[🆕 Новые за последние 30 дней]
[❌ Отмена]
```

Логика сегментов (фильтр по `master_clients`):
- **Все** — все клиенты мастера с `tg_id IS NOT NULL`
- **3+ месяца** — `last_visit < now - 90 days` ИЛИ `last_visit IS NULL`
- **6+ месяцев** — `last_visit < now - 180 days` ИЛИ `last_visit IS NULL`
- **Новые 30 дней** — `first_visit > now - 30 days`

**Шаг 4 — Предпросмотр и подтверждение**
```
📨 Предпросмотр рассылки
━━━━━━━━━━━━━━━
{текст сообщения}
{[медиа если есть]}
━━━━━━━━━━━━━━━
Получателей: 23 клиента
Сегмент: Все клиенты

[📤 Отправить]  [❌ Отмена]
```

Если получателей 0:
```
В этом сегменте нет клиентов с Telegram.
[◀️ Назад]  [🏠 Главная]
```

**После подтверждения — отправка:**
```python
async def send_broadcast(bot, master_id, text, file_id, segment):
    clients = await get_broadcast_recipients(master_id, segment)
    sent = 0
    failed = 0
    for client in clients:
        try:
            if file_id:
                await bot.send_photo(client.tg_id, file_id, caption=text)
            else:
                await bot.send_message(client.tg_id, text)
            sent += 1
            await asyncio.sleep(0.05)  # 50ms пауза — лимит Telegram
        except TelegramForbiddenError:
            failed += 1
        except Exception:
            failed += 1
    return sent, failed
```

После отправки — сохранить в `campaigns` (тип `broadcast`), показать результат мастеру:
```
✅ Рассылка отправлена!

Отправлено: 21
Не доставлено: 2
```

Кнопки: `[🏠 Главная]`

## Критерии проверки

1. Рассылка: все 4 шага FSM работают
2. Медиа прикрепляется и отправляется клиентам
3. Сегментация корректна — каждый сегмент возвращает правильных клиентов
4. Предпросмотр показывает точное количество получателей
5. Если получателей 0 — показывается сообщение, рассылка не запускается
6. Пауза 50мс между сообщениями соблюдается
7. Результат отправки (отправлено/не доставлено) показывается мастеру
8. Акция создаётся, отображается в списке активных
9. Уведомление об акции приходит клиентам (с учётом notify_marketing)
10. Завершение акции убирает её из активных
11. client_bot показывает только активные акции мастера
12. Рассылка сохраняется в campaigns с количеством отправленных
