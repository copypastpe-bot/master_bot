## Правка 1 — Постоянная Reply-кнопка «🏠 Домой» (оба бота)

Добавить постоянную ReplyKeyboardMarkup с одной кнопкой во оба бота.
```python
home_reply_kb = ReplyKeyboardMarkup(
    keyboard=[[KeyboardButton(text="🏠 Домой")]],
    resize_keyboard=True,
    persistent=True  # остаётся видимой всегда
)
```

Отправлять при старте бота (`/start`) вместе с Home-сообщением.

Обработчик в обоих ботах:
```python
@router.message(F.text == "🏠 Домой")
async def handle_home_button(message, state, ...):
    await state.clear()  # прерывает любой FSM
    # удалить сообщение пользователя (саму кнопку)
    await message.delete()
    # показать Home экран
    await show_home(...)
```

**Поведение:**
- Прерывает любой активный FSM (`state.clear()`)
- Удаляет сообщение с кнопкой чтобы не засорять чат
- Возвращает на Home (редактирует постоянное сообщение)
- Работает из любого места — включая середину создания заказа, рассылки и т.д.

## Правка 2 — Завершение сценариев возвратом в родительское меню (оба бота)

После завершения любого FSM-сценария — возвращать в то меню откуда он был запущен, а не на Home.

Правила возврата:
| Сценарий | Возврат после завершения |
|---|---|
| Создать заказ | Экран «Заказы» за выбранный день |
| Провести заказ | Карточка заказа |
| Перенести заказ | Карточка заказа |
| Отменить заказ | Экран «Заказы» |
| Добавить клиента | Экран «Клиенты» |
| Редактировать клиента | Карточка клиента |
| Заметка клиента | Карточка клиента |
| Бонусы клиента | Экран бонусов клиента |
| Рассылка | Экран «Маркетинг» |
| Акция | Экран «Маркетинг» |

| Редактировать профиль | Экран «Профиль» |

**Уведомления** (напоминания за 24ч/1ч, подтверждение записи):
- Оставлять сообщение как есть пока пользователь не нажмёт кнопку
- После нажатия кнопки («Подтверждаю» / «🏠 Домой») — редактировать сообщение, убирать кнопки

## Правка 4 — Выполненные заказы остаются в расписании (master_bot)

**Текущее поведение:** выполненные заказы пропадают из списка.
**Нужное поведение:** показывать выполненные заказы в расписании со статусом.

В списке заказов за день — показывать ВСЕ заказы (все статусы).
На кнопке каждого заказа — эмодзи статуса перед текстом:
```python
STATUS_EMOJI = {
    "new": "🆕",
    "confirmed": "📌",
    "done": "✅",
    "cancelled": "❌",
    "moved": "📅"
}
```

Пример кнопок:
```
✅ 10:00 — Иван П. | Уборка
📌 14:30 — Мария С. | Мойка окон
🆕 18:00 — Алексей К. | Химчистка
```

В карточке выполненного заказа — показывать статус «✅ Выполнен» и убирать кнопки действий (Провести/Перенести/Отменить). Оставить только:
```
[👤 Перейти к клиенту]
[◀️ Назад]  [🏠 Главная]
```

## Правка 5 — Метка «?» на неподтверждённых заказах (master_bot)

Заказы у которых отправлено напоминание за 24ч (`reminder_24h_sent = true`) но клиент ещё не подтвердил — помечать знаком вопроса.

Добавить поле в `orders`:
```sql
ALTER TABLE orders ADD COLUMN client_confirmed BOOLEAN DEFAULT FALSE;
```

Устанавливать `client_confirmed = true` когда клиент нажимает «✅ Подтверждаю запись» (обработчик `confirm_order:{order_id}` в client_bot).

Логика эмодзи на кнопке заказа:
```python
def get_order_emoji(order) -> str:
    if order.status == "done":
        return "✅"
    if order.status == "cancelled":
        return "❌"
    if order.status == "moved":
        return "📅"
    if order.reminder_24h_sent and not order.client_confirmed:
        return "❓"  # уведомлен, не подтвердил
    if order.status == "confirmed" or order.client_confirmed:
        return "📌"
    return "🆕"
```

## Правка 8 — Описание услуги в справочнике (master_bot)

Добавить поле `description` в таблицу услуг:

**Экран услуги в справочнике** — показывать описание если есть:
```
🛠 Уборка квартиры
💰 3 500 ₽
📝 Генеральная или поддерживающая уборка,
   включая санузел и кухню
━━━━━━━━━━━━━━━
[✏️ Название]  [✏️ Цена]
[✏️ Описание]  [📦 В архив]
[◀️ Назад]  [🏠 Главная]
```

**FSM добавления услуги** — добавить шаг «Описание»:
1. Название
2. Цена + `[Без цены]`
3. Описание + `[Пропустить]`
→ Сохранить

**При создании заказа** — если у услуги есть описание, показывать его под названием в шаге выбора услуг (мелким текстом в тексте сообщения, не в кнопке).

## Критерии проверки

1. Кнопка «🏠 Домой» видна постоянно в обоих ботах
2. Нажатие «🏠 Домой» из любого FSM → state.clear() → Home экран
3. После завершения каждого сценария — возврат в родительское меню (не Home)
4. Выполненные заказы видны в расписании с эмодзи ✅
5. Карточка выполненного заказа: статус виден, кнопок действий нет, есть «Перейти к клиенту»
6. Заказ с отправленным напоминанием и без подтверждения — эмодзи ❓
7. После подтверждения клиентом — эмодзи меняется на 📌
8. Услуга: поле описания сохраняется и отображается
9. При добавлении услуги — шаг описания с возможностью пропустить
10. Уведомления не исчезают сами — только после действия пользователя
