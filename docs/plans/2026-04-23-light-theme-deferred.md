# Light Theme — Отложенный план

**Дата:** 2026-04-23
**Статус:** DEFERRED — отложено до завершения пилота

## Контекст

Приложение сейчас принудительно тёмное через `FORCED_THEME` в `main.jsx`.
CSS-переменные `--tg-theme-*` заложены, но переопределяются.
`theme.css` — 3027 строк, ~106 хардкодных `rgba()` dark-specific значений.
~58 хардкодных цветов в JSX-файлах мастер-секции.

## Три пути

### Вариант А — Минимальный «светлый режим» (рекомендован)

- Добавить CSS-класс `body.typeui-enterprise-body.light` с ~25 переопределёнными переменными (фон, карточки, текст, бордеры)
- Убрать `FORCED_THEME` из `main.jsx`, добавить детект темы Telegram через `WebApp.colorScheme`
- Хранить ручное предпочтение в `localStorage`
- Добавить переключатель в страницу «Ещё»
- Компоненты допиливать по мере нахождения проблем

**Время:** ~1 день
**Риск:** часть компонентов может выглядеть некрасиво и потребует дополнительных правок

---

### Вариант Б — Полный аудит двух тем

- Перевести все ~106 хардкодных `rgba()` из `theme.css` в CSS-переменные
- Перевести ~58 inline-цветов в JSX в переменные
- Определить полный набор переменных для dark и light
- Результат идеальный, без регрессий

**Время:** 2–3 дня
**Риск:** низкий, но большой объём работы

---

### Вариант В — Telegram-first без ручного тоггла

- Убрать `FORCED_THEME` из `main.jsx`
- Добавить только CSS-переменные для светлой темы (без аудита компонентов)
- Тема следует автоматически за темой Telegram, ручного переключателя нет

**Время:** ~0.5 дня
**Риск:** при светлой теме Telegram часть компонентов поплывёт
