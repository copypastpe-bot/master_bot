# RU Proxy via Timeweb Moscow Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Дать российским пользователям доступ к Mini App через московский сервер (Timeweb), проксируя трафик на NL (Contabo).

**Architecture:** Два поддомена `ru.app.crmfit.ru` и `ru.api.crmfit.ru` указывают на Timeweb Moscow. nginx на Timeweb отдаёт статику Mini App локально и проксирует API-запросы на NL. Mini App собирается с отдельным `VITE_API_URL`.

**Tech Stack:** nginx, certbot, Vite (build modes), FastAPI CORS

## Task 4: nginx на Timeweb — создать конфиги

**Где:** SSH на Timeweb сервер

**Шаг 1:** Создать конфиг для `ru.app.crmfit.ru` (статика)
```bash
sudo tee /etc/nginx/sites-available/ru.app.crmfit.ru > /dev/null << 'EOF'
server {
    listen 80;
    server_name ru.app.crmfit.ru;

    root /var/www/ru.app.crmfit.ru;
    index index.html;

    # SPA fallback — все пути отдавать index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Кэширование статики
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
EOF
```

**Шаг 2:** Создать конфиг для `ru.api.crmfit.ru` (proxy на NL)
```bash
sudo tee /etc/nginx/sites-available/ru.api.crmfit.ru > /dev/null << 'EOF'
server {
    listen 80;
    server_name ru.api.crmfit.ru;

    location / {
        proxy_pass http://75.119.153.118:8081;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Таймауты (API может отвечать до 30с)
        proxy_connect_timeout 10s;
        proxy_read_timeout 30s;
    }
}
EOF
```

**Шаг 3:** Создать директорию для статики и включить сайты
```bash
sudo mkdir -p /var/www/ru.app.crmfit.ru
sudo ln -sf /etc/nginx/sites-available/ru.app.crmfit.ru /etc/nginx/sites-enabled/
sudo ln -sf /etc/nginx/sites-available/ru.api.crmfit.ru /etc/nginx/sites-enabled/
```

**Шаг 4:** Проверить синтаксис nginx
```bash
sudo nginx -t
```
Ожидается: `syntax is ok` и `test is successful`

**Шаг 5:** Перезагрузить nginx
```bash
sudo systemctl reload nginx
```
