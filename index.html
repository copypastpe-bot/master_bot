Google Integration

Google Calendar Integration

Both bots offer optional Google Calendar integration. When a master or trainer connects their Google account, the bot can automatically create, update, and delete calendar events to reflect their booking schedule.

This integration uses OAuth 2.0 and requests only the minimum permissions required to manage calendar events on behalf of the user.

Permission (Scope) Purpose
calendar.events Create, update, and delete calendar events corresponding to booked client appointments
userinfo.email Display the connected Google account email in bot settings so the user can verify which account is linked

Privacy Policy

Last updated: March 2026

1. Overview

This Privacy Policy describes how our Telegram bots (Master CRM Bot and Fitness CRM Bot) collect, use, and protect user data. By using these bots, you agree to the practices described in this policy.

2. Data We Collect

We collect only the data necessary to provide the service:

  • Telegram user ID, username, and first name (provided by Telegram upon bot interaction)
  • Contact information entered voluntarily: phone number, name, date of birth
  • Order and session data created by the master/trainer
  • Google account email address (only when Google Calendar integration is enabled)
  • Google OAuth tokens (stored securely to maintain Calendar access)

3. How We Use Your Data

  • To operate the bot's core functionality (scheduling, reminders, client management)
  • To sync appointments with Google Calendar when the user has enabled this feature
  • To send automated notifications (appointment reminders, loyalty bonuses)
  • We do not sell, rent, or share personal data with third parties

4. Google API Data Usage

Our use of data obtained via Google APIs complies with the Google API Services User Data Policy, including the Limited Use requirements.

  • Google OAuth tokens are used solely to create, update, and delete calendar events on behalf of the authenticated user
  • We do not use Google user data for advertising or to train machine learning models
  • Access tokens are stored encrypted and are never shared with third parties
  • Users can revoke Google Calendar access at any time via bot settings or through their Google Account security page

5. Data Storage & Security

  • All data is stored on a private server with restricted access
  • OAuth tokens and credentials are stored in encrypted form
  • We do not store payment card data or sensitive financial information
  • Data is retained as long as the user's account is active

6. User Rights

You have the right to:

  • Request deletion of your personal data by contacting us via email
  • Disconnect Google Calendar integration at any time through bot settings
  • Request a copy of the data we hold about you

7. Changes to This Policy

We may update this Privacy Policy from time to time. The current version is always available on this page. Continued use of the bots after changes constitutes acceptance of the updated policy.
